IMPLEMENTACIÓN DEL PATRÓN PROXY PARA LA CLASE CHECKIN
  - EL Gestor_ChekIn (index) FUNGE COMO LA CLASE CLIENTE
  - DECLARAR LA INTERFAZ I_CheckIn CON EL MÉTODO efectua_CheckIn()
  - LA CLASE CheckIn IMPLEMENTA EL MÉTODO efectua_CheckIn() CON LOS PASOS
    REQUERIDOS PARA EJECUTAR EL CHECKIN

  - LA CLASE Proxy_ChekIn TAMBIÉN IMPLEMENTA EL MÉTODO efectua_CheckIn(),
    PERO SU IMPLEMENTACIÓN ES DIFERENTE AL MÉTODO efectua_CheckIn()
    IMPLEMENTADO POR LA CLASE CheckIn(), YA QUE EL Proxy_CheckIn DEBE VALIDAR
    SI ES POSIBLE O NO ejecutar efectua_CheckIn(). SI ES POSIBLE, ENTONCES, 
    INVOCA EL MÉTODO efectua_CheckIn() DE LA CLASE CheckIn. SI NO ES POSIBLE
    (POR EJEMPLO, LA RESERVACIÓN NO HA SIDO PAGADA, O NO FALTAN 48 HORAS O MENOS
    PARA EL VUELO), ENTONCES EL PROXY NO EFECTÚA EL CHECKIN Y REGRESA EL MENSAJE
    CORRESPONDIENTE A LA CLASE GESTOR_CHECKING (index)
